# Higher-Order Markov Chain Music Generator

A production-grade, object-oriented Machine Learning pipeline built in Python to parse classical J.S. Bach compositions and utilize higher-order Markov Chains to generate original, time-duration-controlled MIDI musical sequences.

The project splits the machine learning workflow into two isolated execution phases: the **Training Pipeline** (building and serializing the transition matrix) and the **Inference Pipeline** (loading the model and generating new musical sequences on demand).

# Key Features:
*Higher-Order Constraints:** Configurable Markov Chain orders (n=1, 2, 3, 4) allowing customization of structural phrasing, memory windows, and context retention.

*Massive Corpus Access:** Out-of-the-box native loading and structural parsing of over 400 complete Bach chorales from the robust `music21` library.

*Decoupled Architecture:** Clean Separation of Concerns between data processing, training matrix compilation, serialization (`pickle`), and runtime track generation.

*Smart MIDI Interactivity:** Built-in low-latency automated MIDI synthesizer and playback utilizing an asynchronous, non-blocking listener thread for instant user control.

*Centralized Engineering Config:** Single source of truth config file managing path definitions, model dependencies, execution constraints, and architectural hyperparameters.

🕹️ Workflow & Usage Guide
Step 1: Execute the Training Pipeline (train.py)
This script activates the MusicDataLoader, streams down scores from the music21 digital library, tokenizes note-duration characteristics, builds structural sequence combinations, and saves the binary representation (.pkl) into the artifacts/ folder.
-> python train.py
When to run: Execute this script whenever you change hyperparameters inside src/config.py (such as raising the Markov Order or widening the training size), or when cleaning out old model files.

Step 2: Launch the Inference/Generation System (main.py)
This script skips the heavy training overhead completely, instantly mapping the serialized .pkl file back into system RAM. It generates notes forward based on the learned conditional probabilities, builds a fresh MIDI stream, exports it to your output/ folder with an automated timestamp, and initiates playback.
-> python main.py

Interactive Playback Controls:

As soon as music starts playing, a low-CPU background thread listens for console engagement.

Press [ENTER] inside your terminal at any time to immediately kill the playback engine early without interrupting code execution or causing system hang-ups.



⚙️ Configuration Tuning Panel (src/config.py)
To adjust model behavior, change parameters directly inside src/config.py. Changes will automatically propagate throughout the entire pipeline code:
# Hyperparameters
MARKOV_ORDER = 4        # Context window depth (Higher = more structure, lower = more random)
CORPUS_LIMIT = 400      # Total pieces parsed from music21 (Max 412 Bach chorales)
GENERATION_LENGTH = 200 # Total note tokens generated per run (Larger = longer audio duration)



🎨 Maximizing Audio Realism
MIDI files do not contain recorded sound waves; they contain digitized performance sheets (pitch, time, and dynamic codes).

If the default audio synthesizer provided by your computer sounds basic or electronic, you can achieve cinematic acoustic results by opening your autogenerated timestamped MIDI files (output/gen_*.mid) inside external software:

Digital Audio Workstations (DAW): Import files into platforms like GarageBand, Reaper, Logic Pro, FL Studio, or Cubase and apply high-quality Virtual Studio Technology (VST) instruments like a grand piano or classical pipe organ.

Sheet Music Software: Load tracks directly into free notation editors like MuseScore. This will instantly convert your AI's generated sequence paths into classical sheet music notation while running high-quality acoustic orchestral rendering patches.
